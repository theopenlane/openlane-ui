5. Key Findings

#TypeFinding
1FixedTask detail sheet previously exposed Asset, Campaign, Entity which don't exist on UpdateTaskInput. Removed those 3 types. IdentityHolder kept (it exists in schema).
2FixedControl create form now matches control detail modal: 12 types (added Campaign, Entity, Personnel, Remediation, Review).
3FixedControl detail page now passes all 13 sections to ObjectAssociationSwitch, matching ASSOCIATION_REMOVAL_CONFIG.control. Added entities, identityHolders, campaigns, remediations, reviews to memoizedSections.
4Missing UIProgram has 9 association edges in the schema but no dedicated association modal anywhere. Users can only link programs FROM other entities.
5Missing UICampaign has addControlIDs, addGroupIDs, addIdentityHolderIDs in schema but no association modal.
6Missing UIRemediation has 12 association edges (including Review, Finding, Vulnerability, Risk, etc.) but no association modal at all. Only associated FROM controls.
7Missing UIReview has 11 association edges (including Remediation, Finding, Vulnerability, Risk, etc.) but no association modal at all. Only associated FROM controls.
8InconsistencyEntity (Vendor) schema has addControlIDs but the UI does not expose Control as an association type. Controls can associate to entities but not vice versa.
9InconsistencyIdentity Holder schema has addControlIDs and addFindingIDs but the UI only exposes Asset, Campaign, Task, Vendor.
10Unused edgeActionPlan edges exist on Control, Subcontrol, Risk, Program, Remediation, Review, Task but are never exposed in any UI.
11Unused edgeFinding edges exist on Control, IdentityHolder, Remediation, Review but are never exposed in any UI.
12Unused edgeVulnerability edges exist on Remediation and Review but are never exposed in any UI.
13Unused edgeControlImplementation edges exist on Control, Subcontrol, InternalPolicy, Task but only exposed in Evidence create/detail sheets.
