Grupa A — Okolina (spori next dev server) · 6 testova

Ovi testovi nisu bugovi. Padaju jer se ruta/API kompajlira pri prvom pristupu dulje nego što dopušta timeout. Nestaju s produkcijskim buildom ili „zagrijavanjem" ruta (vidi suggestion.html).

#1 · auth.spec.ts:54

unknown email → generic error, no user enumeration

Greška
TimeoutError: page.waitForResponse: Timeout 10000ms exceeded
  waiting for response /\/api\/auth\/webfinger/
→ zatim: locator.fill timeout na input[name="password"]  (login.ts:24)
Zašto pada

Helper loginViaForm čeka odgovor s /api/auth/webfinger maksimalno 10s. Kod prvog pogađanja te API rute u next dev ona se tek kompajlira i odgovor stigne nakon >10s. Zbog toga se password polje (koje se prikaže tek nakon webfingera) ne renderira i sljedeći fill pukne. Backend i logika su ispravni — kriv je samo cold-compile + pretijesan timeout.
